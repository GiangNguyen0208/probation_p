## YouTube Data API v3 Validation

### Endpoint Strategy

| Endpoint | Routine Sync Use | Quota Note | Decision |
| --- | --- | --- | --- |
| `channels.list` | Yes | Official docs list quota impact as 1 unit. | Use for channel metadata, subscriber count, video count, and view count. |
| `playlistItems.list` | Yes | Official docs list quota impact as 1 unit. | Use uploads playlist for latest video activity instead of search. |
| `search.list` | No, except one-time discovery | Official docs identify a separate default limit for search calls. | Avoid for hourly/daily monitoring sync. |

### Uploads Playlist Rule

Prefer `channels.list?part=contentDetails,statistics,snippet&id=<channel-id>` to retrieve the channel's uploads playlist from `contentDetails.relatedPlaylists.uploads`. Use that playlist ID with `playlistItems.list` for recent uploads. Avoid relying only on the common `UC` to `UU` conversion unless Phase 0 tests prove it is reliable for the target channels.

## YouTube Quota Worksheet

Assumption for routine sync: 1 `channels.list` call plus 1 `playlistItems.list` call per channel per sync cycle.

| Subjects | Sync Frequency | Cycles / Day | Units / Subject / Cycle | Estimated Units / Day | Notes |
| ---: | --- | ---: | ---: | ---: | --- |
| 10 | Hourly | 24 | 2 | 480 | Comfortable for default quota. |
| 50 | Hourly | 24 | 2 | 2,400 | Comfortable if pagination is limited. |
| 100 | Hourly | 24 | 2 | 4,800 | Still viable under a 10,000-unit daily budget. |
| 500 | Every 6 hours | 4 | 2 | 4,000 | Requires staggered scheduling and careful retry policy. |
| 500 | Hourly | 24 | 2 | 24,000 | Not viable without lower frequency, batching strategy, or quota increase. |

Invalid requests still consume quota, so Phase 1 should validate channel IDs before enabling high-frequency scheduled sync.

## Back-Off Requirements

- Treat quota exhaustion and rate-limit responses as retryable only after a cool-down.
- Use exponential back-off such as 1m, 2m, 4m, 8m, then cap at a configured maximum.
- Stop retrying immediately for permanent errors like invalid subject ID, missing channel, or unsupported public data.
- Log platform, endpoint, subject ID, status code, error reason, retry count, and next retry time.

## Future Sandbox Validation Commands

Do not run these commands until sandbox credentials are available in a local `.env` file. Keep real tokens out of committed docs and shell history where possible.

### Facebook Graph API

Load local credentials:

```bash
set -a
source social-data-collector/.env
set +a
```

Validate public Page profile fields:

```bash
curl -G "https://graph.facebook.com/${FACEBOOK_GRAPH_API_VERSION}/${FACEBOOK_TEST_PAGE_ID}" \
  --data-urlencode "fields=id,name,fan_count,followers_count" \
  --data-urlencode "access_token=${FACEBOOK_APP_ACCESS_TOKEN}"
```

Validate public Page posts for activity derivation:

```bash
curl -G "https://graph.facebook.com/${FACEBOOK_GRAPH_API_VERSION}/${FACEBOOK_TEST_PAGE_ID}/posts" \
  --data-urlencode "fields=id,created_time,message,permalink_url" \
  --data-urlencode "limit=10" \
  --data-urlencode "access_token=${FACEBOOK_APP_ACCESS_TOKEN}"
```

Record exact permission errors for denied fields:

```bash
curl -G "https://graph.facebook.com/${FACEBOOK_GRAPH_API_VERSION}/${FACEBOOK_TEST_PAGE_ID}/insights" \
  --data-urlencode "access_token=${FACEBOOK_APP_ACCESS_TOKEN}"
```

### YouTube Data API v3

Load local credentials:

```bash
set -a
source social-data-collector/.env
set +a
```

Validate channel metadata and statistics:

```bash
curl -G "https://www.googleapis.com/youtube/v3/channels" \
  --data-urlencode "part=snippet,statistics,contentDetails" \
  --data-urlencode "id=${YOUTUBE_TEST_CHANNEL_ID}" \
  --data-urlencode "key=${YOUTUBE_API_KEY}"
```

Validate recent uploads using the uploads playlist ID returned by `channels.list`:

```bash
curl -G "https://www.googleapis.com/youtube/v3/playlistItems" \
  --data-urlencode "part=snippet,contentDetails" \
  --data-urlencode "playlistId=${YOUTUBE_UPLOADS_PLAYLIST_ID}" \
  --data-urlencode "maxResults=10" \
  --data-urlencode "key=${YOUTUBE_API_KEY}"
```

Do not use `search.list` for scheduled sync validation. If search is needed for one-time subject discovery, record the business reason and quota budget separately.

## Phase 1 Collector Recommendations

- Build API clients around explicit endpoint methods, not generic URL strings.
- Store raw response samples in test fixtures after removing secrets and unnecessary personal data.
- Make platform field availability explicit in normalizer tests.
- Derive activity frequency from recent public posts/uploads when direct activity fields are unavailable.
- Include a quota guard before scheduled YouTube sync fan-out.
