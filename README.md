# social-api-gateway

Phase 2 service for the Social Intelligence Platform. The single external interface that exposes subject data to the Mini App, Alert Engine, and 3rd-party consumers. Reads from the same Postgres+TimescaleDB database the collector writes to, and uses Redis for cache and rate-limit counters.

## Responsibilities

- REST endpoints under `/v1` for subject reads and activity time-series.
- API key authentication (hashed lookup, constant-time verification, `X-API-Key` or `Bearer`).
- Per-key rate limiting (per-minute and per-day) with `429` and `Retry-After`.
- Redis cache for hot subject reads with TTL-based invalidation; DB fallback on cache miss or error.
- OpenAPI/Swagger documentation auto-generated by FastAPI.
- Structured JSON logs for request lifecycle events.

## Out of Scope (deferred to Sprint 2 of Phase 2)

- `GET /v1/alerts` and `PUT /v1/alerts/:subject_id` (alert rule management).
- Internal-vs-external key tier enforcement on the write path.
- API key self-service / admin endpoints for issuing and revoking keys (keys are created via Alembic seed scripts for now).

## Local Commands

```bash
# Install (from project root)
pip install -e "./social-common[dev]" -e "./social-api-gateway[dev]"

# Apply migrations
cd social-api-gateway
alembic upgrade head

# Run the server
uvicorn social_api_gateway.main:app --reload --host 0.0.0.0 --port 8000

# Or with the configured host/port
python -m uvicorn social_api_gateway.main:app --reload

# OpenAPI / Swagger UI
open http://localhost:8000/docs
open http://localhost:8000/redoc

# Health check
curl http://localhost:8000/v1/health

# Tests
pytest                                            # unit + integration
pytest -m "not integration"                       # unit only
```

## Issuing a Test API Key

For development, seed a key directly via Python:

```python
import asyncio
from uuid import uuid4
from sqlalchemy.ext.asyncio import async_sessionmaker, create_async_engine

from social_api_gateway.auth.models import APIKeyModel, APIKeyTier
from social_api_gateway.auth.security import generate_key, hash_key, key_prefix
from social_api_gateway.config import get_settings
from social_api_gateway.db import Base

async def main():
    settings = get_settings()
    engine = create_async_engine(settings.database.url)
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    SessionLocal = async_sessionmaker(bind=engine, expire_on_commit=False)

    raw = generate_key()
    async with SessionLocal() as session:
        session.add(APIKeyModel(
            id=uuid4(),
            name="Dev Internal Key",
            key_prefix=key_prefix(raw),
            key_hash=hash_key(raw, settings.auth.pepper.get_secret_value()),
            tier=APIKeyTier.INTERNAL,
            is_active=True,
        ))
        await session.commit()

    print(f"API key (save this, it will not be shown again): {raw}")

asyncio.run(main())
```

## Configuration

All configuration is read from environment variables (and a `.env` file at the project root). See the top-level `.env.example` for the full list. The most relevant variables for the gateway:

| Variable | Purpose |
| --- | --- |
| `DATABASE_URL` | async SQLAlchemy URL for the collector's DB |
| `REDIS_URL` | Redis URL for cache and rate-limit counters |
| `API_KEY_PEPPER` | Server-side secret mixed into every key hash. Rotating invalidates every issued key. |
| `GATEWAY_*` | Host, port, workers, API prefix |
| `CACHE_*` | TTLs for subject, list, and activity cache entries |
| `LOG_LEVEL` | Standard structlog level |

## Acceptance Criteria Mapping

| Criterion | Where it lives |
| --- | --- |
| Requests without a valid key are rejected with `401` before business logic | `deps.py:get_api_key` |
| External keys are read-only and rate limited | `rate_limit/service.py` tier config |
| Internal keys can access alert rule endpoints | Deferred to Sprint 2 |
| Cache miss falls back to DB and returns the same response shape as cache hit | `subjects/routes.py` |
| API docs are sufficient for a new integrator to call the API without asking for hidden details | `/docs`, `/redoc`, `/openapi.json` |
